Accessibility statement for the Trust and Academy Management Service (TRAMS) website

TRAMS is run by the Service Delivery Division (SDD) at the Department for Education (DfE).

We want as many people as possible to be able to use this service.

That means you should be able to:

  • change colours, contrast levels and fonts
  • zoom in up to 300% without the text spilling off the screen
  • navigate most of the website using just a keyboard
  • navigate most of the website using speech recognition software
  • listen to most of the website using a screen reader (including the most recent versions of JAWS, NVDA and VoiceOver)

AbilityNet has advice on making your device easier to use if you have a disability.

How accessible this website is

This website is not compliant with Web Content Accessibility Guidelines version 2.1.

You may have problems using:

  • checkboxes and button labels
  • links
  • menus and windows
  • areas where maps or charts are used
  • backgrounds and colours

Non-compliance with the accessibility regulations

We have listed content that is inaccessile here along with any regulation it does not comply with.

  1. Some form elements and controls do not have names that are descriptive to users of assistive technology. This fails WCAG 2.1 success criterion 1.1.1 (Non-text Content) and WCAG 2.1 success criterion 4.1.2 (Name, Role, Value).
  2. Some checkboxes are hidden from screen reading software. This fails WCAG 2.1 success criterion 1.1.1 (Non-text Content) and WCAG 2.1 success criterion 4.1.2 (Name, Role, Value).
  3. There are images with links that do not work with all screen readers. This fails WCAG 2.1 success criterion 1.1.1 (Non-text Content).
  4. Keyboard control is not predictable. This fails WCAG 2.1 success criterion 2.1.1 (Keyboard) and WCAG 2.1 success criterion 2.4.7 (Focus Visible).
  5. Some areas are inaccessible when only using a keyboard. This fails WCAG 2.1 success criterion 2.1.1 (Keyboard).
  6. Although there is an option to 'skip' to the beginning of content, it is only visible to users of screen reading software. This fails WCAG 2.1 success criterion 2.4.1 (Bypass Blocks).
  7. Some content does not contain valid CSS or HTML tags or attributes. This fails WCAG 2.1 success criterion 4.1.1 (Parsing).
  8. When using a keyboard, the tab key does not move the focus in the correct order. This fails WCAG 2.1 success criterion 2.4.3 (Focus Order).
  9. Text becomes obscured or difficult to navigate when zoomed in. This fails WCAG 2.1 success criterion 1.4.4 (Resize text).
  10. Foreground and background colours are not specified. This fails WCAG 2.1 success criterion 1.4.3 (Contrast).
  11. Some elements do not display a focus when using keyboard navigation. This fails WCAG 2.1 success criterion 2.4.7 (Focus visible).

What we're doing to improve accessibility

We are currently gathering information about the needs of our users.

The data we collect will help us decide how to best meet their needs in an accessible way.

We will update this page with our findings, or a predicted date of compliance no later than June 2022.
